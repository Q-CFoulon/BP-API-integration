SecOps Revised Starter Architecture

Identity and Edge

  • Entra ID App Proxy publishes the app with pre-auth.
  • Conditional Access, MFA, and Identity Protection enforced before app access.
  • No inbound public application endpoint required.

Network and Separation

  • Separate Dev and Prod subscriptions and spoke VNets.
  • Shared lightweight hub with NAT Gateway outbound control.
  • One connector in Dev and one to two in Prod for HA as demand grows.

Security and Monitoring

  • Private endpoints on App Service, database, Key Vault, and Storage.
  • Defender for Cloud CSPM and workload protections in both environments.
  • Diagnostics and Entra logs forwarded through Event Hub to Quisitive IT.

Starter Delivery Plan (6 Weeks)

  1. Week 1: Repository, CI and CD baseline, and environment guardrails.
  2. Week 2: Hub and spoke networking, private DNS, and NAT egress.
  3. Week 3: Data plane, Key Vault, and private endpoints.
  4. Week 4: App Proxy apps, connectors, and Conditional Access policies.
  5. Week 5: Port POC workloads to App Service and Function Apps, then enable Defender and forwarding.
  6. Week 6: Smoke tests, failover validation, Quisitive IT log validation, and pilot go-live review.

Exit Criteria to Enterprise Blueprint

  • Tenant count exceeds 10.
  • Active users consistently exceed 10,000.
  • Regulatory pressure requires stronger segmentation or tenant-dedicated stacks.
  • Sustained performance needs larger connector pools or expanded app tiers.
  • SOC automation scope expands to enterprise-scale framework onboarding.
