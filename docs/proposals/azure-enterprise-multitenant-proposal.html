SecOps Revised Architecture Alignment

Identity and Network Pattern

  • Entra ID App Proxy pre-auth, Conditional Access, MFA, and risk policies before workload access.
  • Hub-spoke topology with NAT Gateway egress and private DNS for private endpoint resolution.
  • Dev and Prod separated by subscription, VNet, connector groups, and Entra app registrations.

Application and Data

  • App Service web tier plus Function Apps for automation pipelines.
  • PostgreSQL or Azure SQL for case state and governance records.
  • Blob Storage for evidence, exports, and immutable audit artifacts.
  • Managed identity plus Key Vault plus private endpoints on all critical data plane paths.

Security and Monitoring

  • Defender for Cloud CSPM and workload protections enabled in Dev and Prod.
  • Diagnostic Settings and Entra logs forwarded to Quisitive IT via Event Hub.
  • ISO-aligned controls on access, logging, monitoring, network segregation, and Dev and Prod separation.

Delivery Model (SecOps Revised)

  1. Stand up repository controls and environment approval gates.
  2. Generate and harden IaC and Azure Policy baselines.
  3. Deploy hub-spoke networking, UDRs, NAT, and private DNS.
  4. Provision data and secret services with private access only.
  5. Deploy and register App Proxy connector VMs by environment.
  6. Configure App Proxy enterprise apps and Conditional Access.
  7. Deploy App Service and Function workloads.
  8. Port and validate POC logic on managed Azure services.
  9. Enable Defender, diagnostics, and Event Hub forwarding.
  10. Execute tabletop and failover tests, then approve go-live.
